## Cross-Project Rules

1. **Backend deploys before frontend.** Always verify backend health before deploying frontend changes that depend on new endpoints.
2. **OpenAPI is the contract.** When implementing frontend API calls, fetch the OpenAPI spec first. Never guess endpoint shapes.
3. **Money is int64.** Both repos use minor units (cents/centavos) as integers. No floats anywhere in the stack.
4. **Conventional Commits** in all repos. Same format: `type(scope): description` + `refs fzambone/pfm-workspace#N` (target repos) or `closes #N` (workspace repo).
5. **Each repo's CLAUDE.md is authoritative** for that repo's conventions. This workspace CLAUDE.md adds cross-project coordination rules only.
6. **Auth token is a Bearer JWT.** API clients send `Authorization: Bearer <token>`. The token is obtained via `POST /api/v1/auth/login` and returned as `{"token": "...", "expires_at": "<RFC3339>"}`. Per pfm-go implementation.
7. **All API paths are prefixed `/api/v1/`.** The local spec lives at `api/swagger.yaml` in pfm-go. Do not construct endpoint URLs without consulting the spec.

## Backend Architecture (pfm-go)

Per pfm-go CLAUDE.md and codebase:

- **Architecture:** Hexagonal / ports-and-adapters. Layers: `domain/` → `port/` (interfaces) → `adapter/` (HTTP, DB). No framework.
- **HTTP layer:** `internal/adapter/http/` — one file per domain area (e.g., `account.go`, `user.go`, `ledger.go`). Routes defined in `routes.go`.
- **Middleware:** `internal/middleware/` — `authn` (JWT validation, injects user ID into context), `HouseholdGuard` (membership check), `HouseholdAdminGuard`.
- **Validation:** `internal/platform/validate/` — `validate.NewResult()` collects field violations, returns `*validate.ValidationError`.
- **Persistence:** `internal/adapter/postgres/` — sqlc-generated queries. Migrations via goose.
- **OpenAPI:** Generated by swag from handler comments. Run `make generate` to regenerate `api/swagger.yaml`. Run `make check-openapi` to verify no drift without writing to disk.
- **Test strategy:** Unit tests (table-driven, `t.Run`), integration tests via testcontainers (build tag `integration`). Run `make test-integration` — never `go test -tags integration` directly.
- **Deployment:** Fly.io, app `pfm-go-api`, region `gru`. Deploy with `fly deploy`.
- **Container runtime:** OrbStack (Docker-compatible). Never use `latest` image tags.

## Frontend Architecture (pfm-ui-react)

Per pfm-ui-react CLAUDE.md and codebase:

- **Stack:** React + TypeScript + Vite + Tailwind CSS v4. Package manager: pnpm.
- **Directory layout:** `src/features/` (domain slices), `src/shared/` (cross-cutting utilities), `src/pages/` (route-level components), `src/design-system/` (atomic UI components), `src/api/` (generated or hand-written API client).
- **Architecture:** Feature-based. Each feature owns its components, hooks, and types. No monolithic component files.
- **Styling:** Tailwind utility classes. Class merging via `cn()` / `clsx`. No inline styles without approval.
- **Dependency versions:** Exact — no `^` or `~` for packages that affect behavior. Pin tooling versions.
- **CI gate:** `pnpm ci` — runs lint + type-check + test + build in sequence.

## API Contract Details

- **Base URL (production):** `https://pfm-go-api.zambone.dev/api/v1/`
- **Auth:** `POST /api/v1/auth/login` → `{"token": "...", "expires_at": "..."}`. All protected routes require `Authorization: Bearer <token>`.
- **Content-Type:** Always `application/json` for request and response bodies.
- **Request body limit:** 1 MB (enforced server-side).

### HTTP Error Responses

Per `internal/adapter/http/response.go` in pfm-go:

| Status | When |
|---|---|
| 400 Bad Request | Missing/malformed JSON body, or validation violations (field-level errors) |
| 401 Unauthorized | Invalid credentials (`POST /auth/login`) |
| 404 Not Found | Resource does not exist |
| 409 Conflict | Duplicate (email taken, member already exists) OR optimistic-lock version conflict OR business rule violations (balance not zero, not a credit card account, last admin) |
| 422 Unprocessable Entity | Ledger unbalanced |
| 500 Internal Server Error | Unexpected error (never exposes internal details) |

**Error body format:**
```json
{"error": "human-readable message"}
```

**Validation error body format (400 only):**
```json
{"error": "validation failed", "fields": {"fieldName": "violation message"}}
```

**Success with no body:** 204 No Content (delete, deactivate).
**Created resource:** 201 Created with `Location` header + resource JSON body.

### OpenAPI Drift Check

Run `make check-openapi` (in pfm-go) before any pfm-go PR that touches `internal/adapter/http/`. This runs swag non-destructively and diffs the output against `api/swagger.yaml`. Drift means `make generate` is needed before merging.
